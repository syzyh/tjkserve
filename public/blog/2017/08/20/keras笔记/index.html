keras笔记

发表于 2017-08-20

简介：KerasKeras是一个高层神经网络API，Keras由纯Python编写而成并基Tensorflow、Theano以及CNTK后端。Keras为支持快速实验而生，能够把你的idea迅速转换为结果，具有以下优势：

  1. 简易和快速的原型设计（keras具有高度模块化，极简，和可扩充特性）
  2. 支持CNN和RNN，或二者的结合
  3. 无缝CPU和GPU切换

一些基本概念

符号计算

无论是Theano还是TensorFlow，都是一个“符号主义”的库。

笼统的说，符号主义的计算步骤是首先定义各种变量，然后建立一个“计算图”，计算图规定了各个变量之间的计算关系。建立好的计算图需要通过编译来确定其内部细节，然而，此时的计算图还是一个“空壳子”，里面没有任何实际的数据，只有把需要运算的输入放进去后，才能在整个模型中形成数据流，从而形成输出值。

Keras就是这样，在搭建完模型后，只是一个空壳子，只有实际生成可调用的函数后（K.function),输入数据，才会形成真正的数据流。

张量

张量可以看作向量、矩阵的自然推广。规模最小的0阶张量，即标量，也就是一个数。

当把一些数有序的排列起来，就形成了1阶张量，向量，依次类推。

张量的阶数有时候也称为维度、轴。

泛型模型

之前版本中，模型有两种：Sequential，称为序贯模型，也就是单输入单输出，一条路通到底，层与层之间只有相邻关系，没有跨层连接。这种模型编译速度快，操作上也比较简单。还有一种称为Graph，这个模型支持多输入多输出，层与层之间随便连，但编译速度慢。

新版中图模型被移除，而增加了“functional model API”，一般的模型就称为Model，Sequential作为特殊情况而存在。而“functional model API”作为泛型模型即接受一个或一些张量作为输入，然后输出也是一个或一些张量。

batch

深度学习的优化算法，也就是梯度下降算法，每次更新参数有两种方式：

第一种，遍历全部数据集算一次损失函数，然后算函数对各个参数的梯度，更新梯度。这种方法每更新一次参数都要把数据集里所有样本看一遍，计算量开销很大，称为Batch gradient descent，批梯度下降。

第二种，每看一个数据就算一下损失函数，然后求梯度更新参数，这个称为随机梯度下降，stochastic gradient descent。这个方法速度比较快，但收敛性不太好，容易在最优点附近产生震荡。

现在一般采用一种比较折中的办法，mini-batch gradient decent,小批的梯度下降，把数据分成若干批，按批来更新参数，这样一组数据共同决定本次梯度的方向，下降起来就不容易跑偏，减少了随机性。

现在基本上都是基于mini-batch的，所以模块中经常出现batch_size就是指这个，keras中用的优化器SGD是stochastic gradient descent，但不代表就是一个样本更新一回，还是基于mini-batch的。

列表生成器

通过列表生成器，可以直接创建一个列表，但如果只使用一部分元素，其它内存就被浪费了。

generator保证列表元素是推算出来的，不必创建完整的list，从而节省大量的空间。
