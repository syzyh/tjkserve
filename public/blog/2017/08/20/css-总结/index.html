css 总结

发表于 2017-08-20

简介：假期时间比较清闲，抽时间复习了一下基础知识。以下是阅读《CSS Master》后总结的一些知识点，虽然书是十年前出版，但里面知识体系很完整，依是一本不错的学习css知识的好书。

常用选择器

  1. 元素选择器 p {color: balck;}
  2. 后代选择器：寻找特定元素或元素组的后代。由两个选择器之间的空格表示 blockquote p {padding: 2em;}
  3. id选择器 类选择器 #intro {font-weight: bold;} p.date-posted {color: #ccc;}选中带有特定类的标签

很多开发人员过渡依赖类选择器和ID选择器，更好的方法是结合使用类型、后代、ID和类几种选择器

.main h2 {font-size: 1.8em}

伪类

当需要根据文档结构之外的其他条件对元素应用样式时，例如表单元素或链接的状态。

  1. UI伪类：a:link {color: blue;} a.visited {color: green;} a:hover, a:focus, a:active {color: red;}

tr: hover {backgroun-color: red;} input: focus {background-color: yellow;} p:target {color:red;}

  1. :first-child :last_child表示一组同胞中的第一个元素与最后一个 ol.result li:first-child {color:blue;}

    :nth-child(n)表示第n个元素（n也可以是odd或even）

通过把伪类连接在一起，可以创建更复杂的行为。

比如在已访问链接上实现不同的鼠标悬停效果 a:visited:hover {color: olive;}

通用选择器

通用选择器由一个星号表示，匹配所有可用元素

比如删除每个元素默认的浏览器内边距和外边距: * {padding: 0; margin: 0;}

p * {color:red;}可以把p包含的所有元素的文本变成红色

section * a {font-size:1em;} 可以把section中孙子元素里的a标签选中

高级选择器

  1. 子选择器和相邻同胞选择器：后代选择器选择一个元素的所有后代，而子选择器只选择元素的直接后代，即子元素。相邻同胞选择器定位同一个父元素下某个元素之后的元素 .nav>li {padding: 20px;} h2 + p {color: #777;} h2 ~ a{color:red;} + 是靠近的两个同胞，~是所有的同胞
  2. 属性选择器：对具有某个特定属性的元素应用不同的样式, ~=表示属性中含有一个这个属性值 p[title]:hover {cursor: help;} a[rel=”nofollow”] {padding: 20px;} .main a[rel~=”worker”]
  3. 层叠和特殊性：同一元素可能有多个规则，层叠给每个规则分配一个重要度：作者样式>用户样式>浏览器样式。规则的特殊性最高，作为样式。特殊性分为四个等级：行内样式 > ID选择器的总数 > 类、伪类和属性选择器的数量 > 元素选择和和伪元素选择器数量
  4. 元素标签上添加类或ID body.news{ } body#archive { }

盒模型外边距叠加

当两个或多个垂直外边距相遇时，它们将形成一个外边距。这个外边距的高度等于两个两个发生叠加的外边距的高度重较大者。常见两种情况：垂直位置的上下两个盒的底边距和顶边距之间发生叠加；一个元素包含在另一个元素中时（假设没有内边距或边框隔开），则它们的顶或底边距也会发生叠加。

定位概述position

  1. 相对定位（relative）：元素将出现在它所在的位置上，然后通过设置垂直和水平位置，让元素“相对于”它原本位置的起点移动。使用相对定位时，无论是否移动，元素任然占据原来的空间。移动元素会导致它覆盖其他框。
  2. 绝对定位（absolute）：元素将与文档流无关，不占据空间。其他元素的布局就像绝对定位的元素不存在一样。绝对定位的元素的位置是相对于距离它最近的那个已定位的祖先元素确定的。
  3. 固定定位（fixed）：固定定位是绝对定位的一种，差异在于固定定位的元素是相对于窗口，可以创建总是出现在窗口中相同位置的浮动元素。

浮动与清除

浏览器会把浮动元素往上推，后面的元素会忽略浮动元素，但文本内容会绕开浮动的元素。

正常元素没有定宽时会依据父元素来定。浮动元素一定要有定宽，不然会被压缩。

围住浮动子元素的三种方法：

  1. 为父元素添加overflow：hidden

    overflow:hidden真正的用途是防止包含元素被超大内容撑大，超大子内容依然保持设定的宽度，超出的部分会被容器剪切掉。另外还能迫使父元素包含浮动的子元素。

  2. 同时浮动父元素

    父元素浮动后，不管子元素是否浮动，它都会紧紧包围住它的子元素。另外还需要用width:100%让父元素与浏览器同宽。最后还要保证父元素下方的元素不会挤上去，要给它应用clear:left

  3. 添加非浮动的清除元素

    给父元素的最后添加一个非浮动的子元素，然后清除该子元素。由于父元素一定会包含这个非浮动元素，而且清除会让这个子元素位于浮动元素的下方，以此达到包含浮动元素的效果。

页面布局

布局高度：正常情况下都应该保持元素height属性为auto，这样随着内容增加而在垂直方向上扩展。假如明确设定了元素的高度，那么超出的内容要么被剪掉，要么会跑到容器外—取决于overflow属性的设定。
